# 1С УНФ — Интеграция с заказами

## Статусы заказов для создания в 1С

Статусы в 1С должны соответствовать статусам платформы. Список для ручного создания в 1С УНФ:

| Статус платформы | Название в 1С | Когда используется |
|---|---|---|
| `PROCESSING` | В работе | Клиент подтвердил заказ, заказ поступил из платформы |
| `INVOICE_SENT` | Счёт выставлен | 1С выставила и отправила счёт клиенту |
| `PENDING_PAYMENT` | Ожидает оплаты | Клиент должен оплатить счёт (B2B предоплата) |
| `AWAITING_CONFIRMATION` | Ожидает подтверждения оплаты | Товар отгружен, оплата — после доставки (B2B постоплата) |
| `PAID` | Оплачен | Оплата получена и подтверждена |
| `SHIPPED` | Отгружен | Товар передан в доставку |
| `IN_TRANSIT` | В пути | Товар у перевозчика |
| `DELIVERED` | Доставлен | Клиент получил товар |
| `CANCELLED` | Отменён | Заказ отменён (до отгрузки) |
| `REFUNDED` | Возврат средств | Деньги возвращены клиенту |

> Статусы `DRAFT`, `PAYMENT_FAILED` — внутренние, в 1С не передаются.

---

## Направление синхронизации

**Платформа → 1С:** при переходе в `PROCESSING` (клиент подтвердил заказ)
→ integration-service отправляет CommerceML в 1С

**1С → Платформа:** при смене статуса в 1С (выставлен счёт, оплачен и т.д.)
→ 1С инициирует выгрузку через CommerceML
→ integration-service принимает и обновляет статус в order-service

---

## Задача: настройка автоматической смены статуса в 1С

**Что нужно:** при выставлении счёта клиенту в 1С заказ автоматически переходит в статус
`INVOICE_SENT` и уведомление уходит клиенту на email.

**Что изучить:**
- В 1С УНФ есть "Помощник настройки" (или "Бизнес-процессы" / "Регламентные задания") —
  найти где настраиваются автоматические действия при смене статуса документа
- Конкретно: событие "Счёт на оплату сформирован/отправлен" → триггер → обмен с платформой
- Варианты реализации:
  1. Подписка на событие в 1С + вызов HTTP endpoint integration-service
  2. Регламентное задание: периодически проверяет счета со статусом "Отправлен" и инициирует CommerceML-выгрузку
  3. Обработчик проведения документа "Счёт на оплату"

**Ссылки для изучения:**
- Документация 1С УНФ: "Настройка бизнес-процессов"
- Раздел: Администрирование → Обмен данными → Настройка обмена с сайтом
- Возможно: ИТС (информационно-технологическое сопровождение) → статьи по УНФ + CommerceML
